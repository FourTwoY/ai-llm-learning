# Week 1-2 学习总结

这两周我主要完成了 Python 基础和面向 AI / 大模型开发基础操作的入门学习。

---

## 这两周我学会了什么

### 1. Python 基础语法
我学习了变量、输入输出、基本运算和基础语法规则，并通过简单脚本练习了 Python 的基本使用方法。

### 2. 常用数据结构
我练习了字符串、列表、字典等常见数据结构，能够完成基本的数据存取和遍历操作。

### 3. 函数与循环
我学习了函数定义、参数传递、返回值，以及 `for` / `while` 循环，理解了如何把重复逻辑封装成函数。

### 4. 文件读写
我掌握了文本文件的读取与写入，能够完成简单文件处理任务。

### 5. 模块与代码组织
我开始理解如何把代码拆分到不同文件中，以及如何通过导入模块组织代码。

### 6. 面向对象基础
我接触了类和对象的基本概念，理解了用类描述现实事物的基本方式。

### 7. 虚拟环境与依赖管理
我学习了使用 `venv` 创建虚拟环境，使用 `pip` 安装第三方库，以及用 `requirements.txt` 记录依赖。

### 8. requests 与 JSON 处理
我完成了公开 API 请求练习，能够使用 `requests` 获取 JSON 数据，并提取自己关心的字段。

### 9. pathlib 与文件批处理
我练习了使用 `pathlib.Path` 扫描目录、统计文件信息，并批量输出结果文件。

### 10. pandas 基础
我学习了 `DataFrame`、读取 CSV、选列、计算平均分、保存结果 CSV 等操作，开始接触结构化数据处理。

### 11. 日志、异常处理与代码重构
我开始理解为什么代码不能只满足“能跑”，还要考虑日志、异常处理、函数拆分和可读性。

### 12. 小综合项目实践
我完成了“文档信息整理工具”这个小项目，开始体验一个小工程的基本组织方式，包括：
- 主程序入口
- 工具模块拆分
- requirements.txt
- README.md
- 异常处理
- 日志输出

## 哪些地方还不熟

虽然这两周已经完成了不少练习，但我觉得自己还有这些地方不够熟练：

### 1. 对 Python 工程结构还不够熟
我知道模块拆分和项目目录的重要性，但还不够自然，仍然需要通过实践去强化。

### 2. 对异常处理的理解还不够深
目前我会写 `try-except`，但对于“应该在哪里捕获异常、如何设计异常处理层级”还不够熟练。

### 3. 对 pandas 还只是入门
目前只会基础的读取、选列和简单计算，对过滤、分组统计、缺失值处理还不够熟悉。

### 4. 对 API 返回数据结构的理解还需要加强
面对嵌套 JSON 时，我还需要更多练习，才能更快看懂数据结构并提取信息。

### 5. 对 Git 和 GitHub 的使用还需要继续熟悉
我会基础提交，但对分支、规范提交信息、仓库整理方式还不够熟悉。
